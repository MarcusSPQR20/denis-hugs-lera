В рамках тестового задания необходимо выполнить верстку одностраничного RWD сайта.

Задание состоит из 3х этапов:

  1. Верстка декстопной версии сайта.
  2. Верстка планшетной версии сайта.
  3. Верстка мобильной версии сайта с техническим дизайном, основанным на десктопной версии. Глобальная перестройка лойаута для мобильной версии начинается с 767px.

Дополнительно

  1. Все иконоки необходимо вставлять с помощью svg (желательно использовать спрайт)
  2. Шрифты подключать с Google Fonts
  3. Все элементы которые подразумевают hover/active состояние должны иметь стилизацию для этих состояний.
  4. Sticky header. Пункт меню с выпадающим списком также является ссылкой
  5. Баннер в начале страницы, слайдер изображений, с ф-цией автовоспроизведения. Текст и форма статичны.
  6. В секции 'Onze klanten' заголовок статичен, все остальное слайды переключаемые либо кнопками, либо свайпом
  7. Все ссылки, ведущие на внешние ресурсы должны иметь target _blank

Требования к выполнению работы

  1. Не подключать готовые reset css (необходимо написать собственную)
  2. Различные библиотеки, шрифты необходимо подключать напрямую в проект

Список поддерживаемых браузеров и мобильных устройств

  1. десктопные браузеры: IE9+, FF, Opera, Chrome, Safari – последние версии;
  2. мобильные устройства: iPhone 5+, iPad 3, iPad Air, Samsung Galaxy 3+

Общие требования к верстке

  1. Блочная верстка (div-based) c RWD
  2. Использование любого сборщика проектов (gulp, grunt, webpack)
  3. Чистый html(pug)/scss(sass) код (запрещено использование: javascript (все скрипты выносятся в отдельный файл), expression, style, условных комментариев, не валидых хаков для IE и др.). Исключение: подключение дополнительных корректирующих таблиц для IE.
  4. Кроссбраузерность
  5. Валидный код HTML и CSS для HTML5/CSS3 (http://validator.w3.org)
  6. Минимальный оптимизированный html/css код. Использование идентификаторов (id) только для создания «каркаса» верстки, работа с классами (class), создание базовых классов.
  7. Все оформление и изображения, не изменяемые динамически, выносятся в таблицу стилей (css). Ничего не должно быть прописано через inline стили. Исключением могут быть стили динамически добавляемые javascript.

Общие требования к оформлению кода

1.В коде (html/css) не должно быть неиспользуемых блоков, свойств, закомментированных участков, нереализованных мыслей и пр. «мусора».

2.Названия идентификаторов и классов должны иметь значимые либо соответствующие месту применения имена на английском языке (н-р: запрещается использование классов типа .gyh1, .a1, .tochecka, .risunok, . logo_search, .topMenu3 и им подобных). Все имена классов только в нижнем регистре (н-р: .sidebar, .logo, .shopping-cart). Разрешено использовать систему сокращений (н-р: классы для таблиц, содержащих информацию – .tbl-«table-name»).

3.Желательно не использовать float.
